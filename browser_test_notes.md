# 브라우저 검증 노트

## 1차 검증

초기 브라우저 진입 시 `RangeError: Maximum call stack size exceeded`가 발생했다. 원인은 BEP 계산을 위해 `calculateRevenue()` 내부에서 다시 `calculateRevenue()`를 호출하는 재귀 구조였다. 해당 부분을 별도의 `profitAt()` 계산식으로 분리해 수정했다.

## 2차 검증

수정 후 브라우저에서 삼첩분식 v1 시뮬레이터가 정상 렌더링되는 것을 확인했다. 히어로, 1·2·3첩 안내 카드, 매출 시뮬레이션 입력 패널, 손익 리포트, 채널 믹스, 상위 매장 벤치마크, 개설비용 시뮬레이션, 자료 기준 섹션이 모두 표시되었다.

| 검증 항목 | 결과 |
|---|---|
| 페이지 제목 | 삼첩분식 v1 매출·개설비용 시뮬레이터 표시 |
| 히어로 디자인 | 레드·옐로우 패키지 콜라주 이미지 및 큰 헤드라인 정상 표시 |
| 매출 기본값 | 월매출 3,234만원, 예상 영업이익 888만원, BEP 1,006만원 표시 |
| 개설비용 기본값 | 12평, 플렉스 간판, 홀 겸업형 기준 실납부 예상 2,583만원 표시 |
| 런타임 오류 | 2차 검증에서 오류 없음 |

## 빌드 검증

`pnpm run check`와 `pnpm run build`를 실행했으며 모두 통과했다. Vite 번들 크기 경고가 있으나, 현재 정적 프론트엔드의 동작을 막는 오류는 아니다.

## 입력 상호작용 검증

월매출 입력값을 2,500만원으로 변경했을 때 손익 리포트가 즉시 갱신되는 것을 확인했다. 예상 영업이익은 595만원, 영업이익률은 23.8%로 표시되었고, 월매출 카드와 비용 막대가 변경된 매출 기준으로 업데이트되었다.

| 상호작용 | 확인 결과 |
|---|---|
| 월매출 입력 변경 | 3,234만원에서 2,500만원 기준으로 손익표 갱신 |
| 예상 영업이익 표시 | 595만원 표시 확인 |
| 계산 연동 | 물류비, 플랫폼·배달 수수료, 예상 주문수 갱신 확인 |

## 개설비용 상호작용 검증

본사비용 면제 프로모션 체크를 해제했을 때 개설비용 리포트가 정상가 기준으로 갱신되는 것을 확인했다. 12평, 플렉스 간판, 홀 겸업형, 2인 테이블 6개 기준에서 실납부 예상 투자비가 2,583만원에서 3,373만원으로 변경되었고, 프로모션 차감액은 0원으로 표시되었다.

| 상호작용 | 확인 결과 |
|---|---|
| 본사비용 면제 체크 해제 | 정상가 기준 총액 3,373만원 표시 |
| 프로모션 차감액 | 0만원 표시 |
| 항목별 리포트 | 본사비용 정상가, 인테리어, 간판, 주방설비 등 항목 정상 표시 |
