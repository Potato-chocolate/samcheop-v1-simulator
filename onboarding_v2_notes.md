# v2(단막) 코드베이스 온보딩 노트

## 1. 원본 위치와 복사본

사용자 데스크톱의 `단막 가맹 매출 및 개설비용 모의계산` 폴더에는 `v1(단막)`, `v1(삼첩분식)`, `v2(단막)` 폴더가 존재한다. 상위 및 현재 폴더에서 `AGENTS.md`는 발견되지 않았고, 대신 `CLAUDE.md`가 있어 기존 프로젝트 설명과 핵심 계산 구조를 확인했다. 분석을 위해 원격 폴더를 압축한 뒤 `/home/ubuntu/onboarding_sources` 아래에 복사했다.

## 2. v2(단막)의 성격

v2(단막)는 **순수 HTML/CSS/Vanilla JS 기반의 단일 페이지 계산기 2종**으로 구성되어 있다. 배포용 파일은 `deploy/cost.html`, `deploy/revenue.html`이며, 계산 데이터는 각각 `deploy/config-cost.js`, `deploy/config-revenue.js`에 분리되어 있다. React 프로젝트로 이식할 때는 이 구조를 `data/config`, `calculation utilities`, `page components`로 나누는 것이 적합하다.

| 구분 | v2 파일 | 역할 |
|---|---|---|
| 개설비용 | `deploy/cost.html` + `config-cost.js` | 평수, 테이블 수, 본사비용, 인테리어, 가구, 화로, 주방설비, 별도공사를 계산 |
| 매출 | `deploy/revenue.html` + `config-revenue.js` | 지역, 상권, 평수, 영업일수, 임대료를 받아 월매출·비용·순수익 계산 |
| 문서 | `PLAN.md`, `CLAUDE.md` | v2 변경사항, 데이터 테이블, UI/UX 확정사항, 계산 흐름 설명 |

## 3. v2 공통 설계 원칙

v2는 각 계산기마다 모든 CSS와 DOM이 HTML에 들어 있고, 숫자 데이터만 외부 JS 설정 파일로 분리되어 있다. 데이터 테이블이 `CONFIG` 객체에 집중되어 있어 v1(삼첩분식) 자료를 반영할 때는 **CONFIG 값 교체 → 계산식 검증 → UI 레이블 교체** 순서로 접근하면 된다.

## 4. v2 개설비용 계산 핵심 구조

개설비용은 `updateStoreInfo()`에서 평수와 테이블 수를 갱신하고, `updateCostItems(area, tables)`에서 각 비용 항목을 다시 계산한 뒤, `calculateGrandTotal()`이 섹션 합계를 모아 최종 금액을 갱신한다. 주요 연동은 다음과 같다.

| 영역 | v2 로직 요약 |
|---|---|
| 평수→테이블 수 | `tablesByArea` 기준으로 30평 12개, 40평 16개, 50평 20개 등 자동 세팅 |
| 인테리어 | `interiorPriceByArea[평수] × 평수` |
| 테이블·의자 | 4인/6인 테이블 및 의자 수량이 좌석 수 기준으로 동기화 |
| 화로 | 테이블 수에 따라 화로세트, 불판(`테이블×3`), 직화기 수량 자동 계산 |
| 착화기 | 35평 이하 1대, 40평 이상 2대 |
| 주방설비·집기 | 평수별 총액 테이블(`equipmentByArea`, `fixtureByArea`) 사용 |
| 별도공사 | 덕트, 단지, 에어컨, 음향, 디자인물, 철거, 도시가스 등 체크박스 기반 선택 합산 |

## 5. v2 매출 계산 핵심 구조

매출은 `updateCalculation()`에서 `테이블 수 × 조정 조단가 × 기대회전수 × 영업일수`로 월매출을 산출하고, `updateProfitCalculation()`이 비용 항목을 계산해 순수익을 갱신한다. 주요 항목은 다음과 같다.

| 영역 | v2 로직 요약 |
|---|---|
| 매출 공식 | `tablesByArea[평수] × areaTypeData[상권].price × turnover × 영업일수` |
| 지역 조정 | 지방권 선택 시 조단가 조정 로직 존재 |
| 인건비 | 등록 평수는 직접값, 미등록 평수는 선형 보간법 사용 |
| 물류비 | 식재료, 주류음료, 수도광열비, 부자재를 매출 대비 비율로 계산 |
| 카드수수료 | 연매출 10억 미만 1.1%, 이상 1.4%, 카드결제 비율 95% 기준 |
| 포스 | `(테이블수+1) × 테이블당 비용 + 기본비` 형태 |
| 고정비 | 임대료, 캐치테이블, 보안, 인터넷, 보험, 기장대행 등 |
| 기타비 | 로열티, 단지렌탈, 카드수수료, 홍보비, 복리후생비 |

## 6. v1(삼첩분식) 이식 시 예상 작업

v1(삼첩분식)에는 현재 Excel 자료 2개가 확인된다. 하나는 가맹가이드 성격의 파일이고, 하나는 `26평 매출분석시트_26.03(제작중)` 파일이다. 다음 단계에서는 두 파일의 시트 구조, 셀 데이터, 공식, 병합 셀, 금액 단위를 추출하여 v2의 `CONFIG` 구조에 맞는 삼첩분식용 데이터 모델을 정의해야 한다.

## 7. 구현 방향 메모

새 프로젝트는 React 19 + Tailwind 기반의 정적 프론트엔드로 준비되어 있다. v2의 단일 HTML 구조를 그대로 복사하지 않고, 다음과 같이 분리해 구현한다.

| 모듈 | 제안 역할 |
|---|---|
| `data/samcheopConfig.ts` | 삼첩분식 기준 평수, 상권, 비용, 비율, 기본값 |
| `lib/simulation.ts` | 매출·비용 계산 순수 함수 |
| `components/SimulatorShell.tsx` | 입력 패널, 결과 패널, 설명 패널 공통 레이아웃 |
| `pages/Home.tsx` | 매출/개설비용 탭 또는 분할 워크벤치 화면 |

선택한 디자인 철학은 **한국형 네오 브루탈리즘 기반의 프랜차이즈 실무 대시보드**이며, 모든 UI는 입력·계산·해석이 명확히 분리되는 상담용 워크벤치 형태로 설계한다.
