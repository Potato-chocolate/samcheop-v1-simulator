# meta

## Purpose
drizzle-kit's internal bookkeeping for the migration history — the journal and one JSON snapshot per migration. These files are written by `drizzle-kit generate` and read by `drizzle-kit migrate`; together they let the tool diff the current `schema.ts` against the previous shipped state to produce the next migration.

## Key Files
| File | Description |
|------|-------------|
| `_journal.json` | Migration ledger — ordered list of `(idx, version, when, tag, breakpoints)` entries that drizzle-kit consumes |
| `0000_snapshot.json` | Schema snapshot captured at the time of migration `0000_zippy_doctor_faustus.sql` |
| `0001_snapshot.json` | Schema snapshot captured at the time of migration `0001_stiff_bucky.sql` |

## For AI Agents

### Working In This Directory
- **Never hand-edit these files.** They must be regenerated by drizzle-kit (`pnpm db:push`) when the schema changes — otherwise the diff against the next migration is corrupted and you risk silent data-shape drift.
- Commit snapshot + journal updates **together with** the matching `../<index>_*.sql` file in a single commit.
- If a snapshot/journal mismatch breaks `drizzle-kit migrate`, the safest recovery is to delete the offending migration and snapshot, fix the schema, and regenerate from scratch — not to patch the JSON.

### Testing Requirements
- None. Integrity is enforced by drizzle-kit at generate/migrate time.

## Dependencies

### Internal
- `../schema.ts` — source of truth that drizzle-kit diffs against the latest snapshot
- `../<NNNN>_*.sql` — generated migration that pairs 1:1 with `<NNNN>_snapshot.json`

### External
- `drizzle-kit` is the only writer of these files.
